# Linux/Kubernetes/OpenShift 状態監視システム 要件整理

## 1. システム概要

Windows上で、企業内で承認済みのPython実行環境を使って動作するサーバ
アプリケーションを起動し、Webブラウザから、踏み台サーバ経由でLinux/Kubernetes/
OpenShift環境の稼働状態を確認できるダッシュボードを提供する。

未署名・不明なexeファイルの実行はEDR/アプリケーションホワイトリスト等で
ブロックされる可能性があるため、単体の実行ファイル（.exe）は作成せず、
**承認済みPythonインタプリタでスクリプトを実行する形**を採用する。

## 2. ネットワーク構成

```
[Webブラウザ] --HTTP--> [Windows: Pythonサーバアプリ（python.exeで実行）]
                              │
                              │ SSH（常時トンネル維持・ID/パスワード認証）
                              v
                        [踏み台サーバ]
                              │
                              │ SSH（多段接続・秘密鍵認証：パスフレーズなし）
                    ┌─────────┼─────────────┐
                    v                       v
        [コマンド実行用VM]         [各種Linux VM群]
         kubectl / oc              各種サービス（設定で追加可能）
         + kubeconfig 設定済み
                    │
                    v
        [Kubernetes/OpenShiftクラスタ]
```

- Windows → 踏み台サーバ：**ID/パスワード認証**
  - パスワードの安全な保管方法を別途検討（例: Windows DPAPIによる暗号化、
    起動時入力方式など。設定ファイルへの平文保存は避ける）
- 踏み台サーバ → 各対象サーバ：**SSH秘密鍵認証**（暗号化済み、パスフレーズ不要）
- 接続方式：**常時SSHトンネルを維持**する多段接続方式
  （踏み台への1本のSSH接続上でチャネルを都度張り、各対象サーバへ到達する想定）

## 3. 監視対象と取得情報

| 対象 | 経路 | 取得コマンド／情報 |
|---|---|---|
| Kubernetes/OpenShiftクラスタ | 踏み台 → コマンド実行用VM | 共通: ノード一覧・Pod一覧（`kubectl get nodes`/`oc get nodes`、`kubectl get pods -A`/`oc get pods -A`）。OpenShiftの場合はこれに加えてClusterOperator（`oc get co`）・MachineConfigPool（`oc get mcp`）も取得 |
| 各種サービス稼働VM | 踏み台 → 対象VM | サービスごとに任意のコマンド・確認内容を設定で追加（例: サービスの稼働状態、ステータス確認コマンドの結果など） |

`oc`コマンドは`kubectl`と互換性のあるサブコマンドを含むスーパーセットのため、
素のKubernetes環境では`kubectl`ベースの基本チェックのみを、OpenShift環境では
それに加えてOpenShift固有のチェックを実行する、という構成で両対応する。

対象VMで実際にどのようなサービスが動作しているか（クラスタ管理ソフト、ロードバランサ、
監視系サービス等）に応じて、`sudo`権限が必要なコマンドを実行するケースもあるため、
NOPASSWD設定など事前の権限設定が前提となる。

対象サーバの種別（OpenStack VM／Linux VM／Kubernetes・OpenShiftノード）ごとに
実行するチェック内容が異なるため、**サーバ（ロール）ごとにチェック項目を
定義できる設定の仕組み**が必要。

## 4. 状態更新（ポーリング）仕様

- デフォルト更新間隔：**1分程度**
- 更新間隔は**設定可能**（設定ファイルまたはUI上で変更）
- **自動更新の一時停止**機能
- 一時停止中も含め、**手動更新ボタン**でいつでも再取得可能

## 5. Web画面の認証

- **認証不要**（社内LAN限定運用を想定）
